# Assess Skill Level - Success Checklist

## Pre-Workflow

- [ ] User is comfortable and not feeling judged
- [ ] Conversational tone established (chat, not test)
- [ ] User confirmed they're ready to proceed

## During Assessment

- [ ] Programming fundamentals assessed (1-3 scale)
- [ ] Problem-solving approach discussed
- [ ] Code organization assessed (1-3 scale)
- [ ] Testing & debugging practices assessed (1-3 scale)
- [ ] Architecture thinking assessed (1-3 scale)
- [ ] Domain knowledge assessed (1-3 scale)
- [ ] Learning goals captured (free text)
- [ ] Coaching preferences captured (free text)
- [ ] User given space to elaborate if desired

## Scoring & Calculation

- [ ] Average score calculated from numeric ratings
- [ ] Overall level determined (beginner/intermediate/advanced)
- [ ] Strengths identified (2-3 areas)
- [ ] Growth opportunities identified (1-2 areas, framed positively)
- [ ] Coaching adaptations determined based on scores

## Summary & Confirmation

- [ ] Summary presented to user
- [ ] User confirmed summary feels accurate
- [ ] Any adjustments made based on user feedback
- [ ] Next steps recommended

## Documentation

- [ ] Assessment report created at `{output_file}`
- [ ] Morgan's memories.md updated with:
  - [ ] Overall level
  - [ ] All dimension scores
  - [ ] Learning goals
  - [ ] Coaching preferences
  - [ ] Coaching adaptations
- [ ] Assessment document includes:
  - [ ] All scores and ratings
  - [ ] Strengths and growth opportunities
  - [ ] Learning goals
  - [ ] Recommended workflows
  - [ ] Reassessment guidance

## Post-Workflow

- [ ] User knows where to find assessment report
- [ ] User knows how to reassess in the future
- [ ] User has 2-3 recommended next workflow options
- [ ] Positive, encouraging tone maintained throughout

## Quality Checks

- [ ] User felt supported, not judged
- [ ] Assessment feels accurate to user
- [ ] Coaching adaptations are actionable
- [ ] Growth opportunities framed positively
- [ ] User understands how Morgan will adapt
- [ ] User feels excited/confident about next steps

## Common Issues to Avoid

- ❌ Making user feel tested or graded
- ❌ Being too formal or rigid
- ❌ Not giving user space to elaborate
- ❌ Framing weaknesses negatively
- ❌ Moving too fast through questions
- ❌ Not confirming summary with user
- ❌ Forgetting to save to memories.md

## Morgan's Reminders

- Be warm, curious, and supportive throughout
- This is a conversation, not an interrogation
- Let them talk - you're learning about THEM
- Frame everything with growth mindset
- Celebrate strengths, be gentle with growth areas
- Adapt your energy to match theirs
- Make sure they feel good at the end
